Local-first privacy

A media editor that avoids backend upload.

Obscura is designed for personal creators who want quick image and short-video edits without sending private files to a cloud workspace.

Open editor Private image tools

What stays local in v1.

Uploaded files are represented by browser File objects, object URLs, and session state. The first MVP does not include accounts, cloud storage, server uploads, or remote media processing.

Selected files

The app works with files that you explicitly choose or drop into the workspace.

Edit state

Crop values, subtitle cues, and export settings live in the browser session.

Generated results

Processed images or videos become temporary local session assets before download.

Where network requests can still happen.

A local-first app can still download application assets. Background removal may load model/runtime files, and the static web app loads JavaScript, CSS, images, and WASM assets. These are app resources, not uploads of your selected media.

Any future cloud processing, account system, remote persistence, or analytics over user media requires a source-of-truth update before implementation.

Local-first FAQ.

Does Obscura store raw media long term?

No. Raw media persistence is not part of the v1 product boundary.

Does Obscura have accounts?

No. Accounts, sync, and collaboration are outside the first MVP.

Why not use cloud editing?

Many personal creators need quick edits for private files and do not want an upload step.
